Integration Roadmap

Features that connect to the Tech Portal ‚Äî what's built, what's ready to plug in, and what's planned.

üîí
Privacy Controls
Built
Company owners control what customer data techs see. Email, phone, and notes can each be toggled independently. Address is always visible for navigation. Privacy filter is enforced server-side in the API route ‚Äî not just hidden in the UI.
companies.settings.techPortal API-level enforcement Per-company config
üé´
Invite / Revoke Flow
Built
Admin clicks "Enable Portal Access" on any team member with technician role. Creates a Supabase auth account, links it to the team member record, and shows a one-time temporary password. Revoke removes auth access completely ‚Äî deletes the auth user.
Admin client (service role) team_members.user_id One-time password display
üóìÔ∏è
Job Assignment + Schedule
Built
Jobs are assigned via a team member dropdown (UUID, not free text). The tech schedule fetches jobs WHERE assigned_to = teamMemberId. Techs can update status to "In Progress" or "Completed". Date tabs: today, tomorrow, this week.
jobs.assigned_to ‚Üí team_members.id Status: scheduled ‚Üí in_progress ‚Üí completed Date range filtering
üß≠
Google Maps Navigation
Built
"Navigate to Address" button on job detail opens Google Maps with the job address pre-filled. Works on mobile ‚Äî opens the Maps app directly for turn-by-turn navigation. Uses the customer address or the job-specific address if different.
maps.google.com deep link Mobile app handoff
üõ°Ô∏è
Middleware Role Routing
Built
Middleware detects technician role on login and redirects to /tech. Techs are blocked from accessing any dashboard route (/customers, /settings, etc.). Admins can access both portals. This is a hard security boundary, not just UI navigation.
middleware.ts team_members.role check Auth redirect
üì∏
Job Photos
Ready to Plug In
The media feature already has MultiImageUploadField, useBunnyUpload, and JobPhotosSection for the admin dashboard. To integrate: add a photo upload section to TechJobDetail so techs can document before/after work on-site. Uploads go to Bunny CDN, same as admin photos.
features/media/ Bunny CDN upload Before/after photos ~50 lines to integrate
üìç
Multi-Stop Routes
Ready to Plug In
The job-stops feature already has StopsEditor and useJobStops. To integrate: show a read-only stop list on TechJobDetail with checkboxes for techs to mark stops complete as they progress through the route. The "Navigate" button could cycle to the next uncompleted stop.
features/job-stops/ Stop progression tracking Next-stop navigation
üè∑Ô∏è
Custom Fields on Jobs
Ready to Plug In
The custom-fields feature renders dynamic forms for any entity type. To integrate: show job custom fields (read-only) on TechJobDetail ‚Äî e.g., "Equipment Needed", "Pool Type", "Access Instructions". Companies define their own fields per their workflow.
features/custom-fields/ CustomFieldsForm component Read-only display for techs
üîÑ
Recurring Job Indicator
Ready to Plug In
Jobs from recurring templates have isRecurringTemplate: true and recurrenceConfig. To integrate: show a subtle "Recurring" badge on the job card and schedule so techs know this is a regular visit. Helps them know the customer and site.
features/recurring-jobs/ Badge on TechJobCard Frequency display
‚è±Ô∏è
Time Tracking
Future
Timer that starts when tech taps "Start Job" and stops on "Complete Job". Records actual time spent per job. Visible to admins in reports. Could include break tracking and travel time. Needs a new job_time_entries table.
New DB table Timer UI component Admin reporting Break tracking
‚úçÔ∏è
Completion Notes + Signature
Future
Techs add notes when completing a job (what was done, parts used, issues found). Optional customer signature capture on a canvas element. Notes flow back to the admin's job detail view.
Canvas signature capture Completion form modal Admin visibility
üìä
Tech Performance Dashboard
Future
Admin view showing per-technician metrics: jobs completed this week/month, average time per job, on-time rate, customer ratings. Helps owners optimize scheduling and identify training needs.
Admin reports page Aggregated metrics Charts/graphs

In-App Messaging
Future
Simple messaging between admin/office and techs in the field. Could leverage Supabase Realtime for instant delivery. Keeps communication within the app instead of personal text messages.
Supabase Realtime Push notifications Message threads
